# Version 5.4.0 (XX/03/2025)

## Changements 

[CORRECTION]
* La liste des agents remontés pour les entretiens professionnels dans les structures considère maintenant la période en la date de prise de poste et la date de fin de campagne.
* Ajout d'un bandeau sur les écrans liés aux fiches de poste


## Modification de la base de données

### Sans BddAdmin

```postgresql
alter table entretienprofessionnel add convocation timestamp;

create table entretienprofessionnel_campagne_progression
(
    id                    serial
        constraint entretienprofessionnel_campagne_progression_pk
            primary key,
    campagne_id           integer                 not null
        constraint epcp_entretienprofessionnel_campagne_id_fk
            references entretienprofessionnel_campagne
            on delete cascade,
    structure_id          varchar(1024)
        constraint epcp_structure_id_fk
            references structure
            on delete cascade,
    nb_total              integer   default 0     not null,
    nb_complet            integer   default 0     not null,
    nb_autorite           integer   default 0     not null,
    nb_observation        integer   default 0     not null,
    nb_superieur          integer   default 0     not null,
    nb_planifier          integer   default 0     not null,
    nb_manquant           integer   default 0     not null,
    date_refresh          timestamp default now() not null,
    histo_creation        timestamp default now() not null,
    histo_createur_id     integer   default 0     not null
        constraint epcp_unicaen_utilisateur_user_id_fk
            references unicaen_utilisateur_user,
    histo_modification    timestamp,
    histo_modificateur_id integer
        constraint epcp_unicaen_utilisateur_user_id_fk_2
            references unicaen_utilisateur_user,
    histo_destruction     timestamp,
    histo_destructeur_id  integer
        constraint epcp_unicaen_utilisateur_user_id_fk_3
            references unicaen_utilisateur_user
);
```
