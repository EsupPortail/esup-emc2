## Modification des données

```
-- Nouveaux paramètres
INSERT INTO unicaen_parametre_parametre(CATEGORIE_ID, CODE, LIBELLE, DESCRIPTION, VALEURS_POSSIBLES, VALEUR, ORDRE)
WITH d(CODE, LIBELLE, DESCRIPTION, VALEURS_POSSIBLES, VALEUR, ORDRE) AS (
    SELECT 'CREP_BLOCAGE_VISUALISATION_AGENT_AVANT_VALIDATION', 'Blocage de l''affichage du CREP aux agents sans validation du responsable', 'Si le paramètre est à <i>true</i> alors les agents ne peuvent pas visualiser le CREP avant la validation de leur responsable', 'Boolean', false, 5 UNION
    SELECT 'CREF_BLOCAGE_VISUALISATION_AGENT_AVANT_VALIDATION', 'Blocage de l''affichage du CREF aux agents sans validation du responsable', 'Si le paramètre est à <i>true</i> alors les agents ne peuvent pas visualiser le CREF avant la validation de leur responsable', 'Boolean', false, 6
)
SELECT cp.id, d.CODE, d.LIBELLE, d.DESCRIPTION, d.VALEURS_POSSIBLES, d.VALEUR, d.ORDRE
FROM d
JOIN unicaen_parametre_categorie cp ON cp.CODE = 'ENTRETIEN_PROFESSIONNEL';
```

## Évolution de la liste des paramètres

Deux nouveaux paramètres pour le module d'entretien professionnel
- `CREP_BLOCAGE_VISUALISATION_AGENT_AVANT_VALIDATION` qui bloque si à la valeur 'true' l'affichage du CREP si la validation du supérieur·e n'est pas faite
- `CREF_BLOCAGE_VISUALISATION_AGENT_AVANT_VALIDATION` qui bloque si à la valeur 'true' l'affichage du CREF si la validation du supérieur·e n'est pas faite
Ils sont initialisés à la valeur 'false' pour ne pas provoquer de changement vis-à-vis des versions 5.3.X.
